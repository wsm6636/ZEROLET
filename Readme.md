# The zeroLET Task Model and its Application to Offset Design Space Exploration

This repository reproduces the evaluation from

*"The zeroLET Task Model and its Application to Offset Design Space Exploration". Shumo Wang, Enrico Bini, Qingxu Deng, Martina Maggio*

The current codebase contains the C implementation of the zeroLET analysis and
evaluation programs. Python is only used by the optional plotting helper.

## Environment Setup

### Requirements

To build and run the C evaluation programs, install a C compiler:

- Windows: MinGW-w64 GCC, MSYS2 GCC, or another C11-compatible compiler
- Linux: GCC or Clang

The runtime evaluation writes CSV and SVG files directly from C. If you want to
generate PNG plots from an existing CSV file, Python 3 with `pandas` and
`matplotlib` is optional:

```bash
pip install pandas matplotlib
```

### File Structure

```text
├── data_c                               # Experiment results generated by C programs
├── analysis_zero_let.c                  # zeroLET analysis implementation
├── analysis_zero_let.h                  # zeroLET analysis declarations
├── evaluation_zero_let.c                # Serial runtime evaluation and shared evaluation helpers
├── evaluation_zero_let_linux_parallel.c # Linux pthread-based parallel runtime evaluation
├── extremes_zero_let.c                  # Extremes evaluation entry point
├── plot_zero_let_csv.py                 # Optional helper for PNG plots from CSV files
└── Readme.md
```

### Deployment

Clone the repository or download the
[zip file](https://github.com/wsm6636/ZEROLET/archive/refs/heads/main.zip):

```bash
git clone https://github.com/wsm6636/ZEROLET.git
cd ZEROLET
```

## How to Run the Experiments

All generated experiment outputs are stored in `data_c/`.

### Extremes Evaluation

Compile:

```bash
gcc -std=c11 -Wall -Wextra -O2 -DZEROLET_EVALUATION_NO_MAIN analysis_zero_let.c evaluation_zero_let.c extremes_zero_let.c -lm -o extremes_zero_let_c
```

Run with default settings:

```bash
./extremes_zero_let_c
```

Run with custom parameters:

```bash
./extremes_zero_let_c 2 12 3
```

Arguments:

| Argument | Description | Default |
|---|---|---|
| `perioddown` | minimum task period | `2` |
| `periodup` | maximum task period | `12` |
| `num_chains_extreme` | task chain length for extremes evaluation | `3` |

To reproduce the **Table IV** results reported in the paper, use
`perioddown = 2`, `periodup = 12`, and `num_chains_extreme = 3`:

```bash
./extremes_zero_let_c 2 12 3
```


### Parallel Runtime Evaluation

On Linux, you can build the pthread-based parallel runtime evaluator:

```bash
gcc -std=c11 -Wall -Wextra -O2 evaluation_zero_let_linux_parallel.c analysis_zero_let.c -lm -pthread -o evaluation_zero_let_linux_parallel
```

Run:

```bash
./evaluation_zero_let_linux_parallel 3 6 1
```

Arguments:

| Argument | Description | Default |
|---|---|---|
| `num_limit` | minimum task chain length | `3` |
| `num_chains` | maximum task chain length | `6` |
| `num_repeats` | repetitions for each chain length | `1` |


To reproduce the **Figure 8** results reported in the paper, use
`num_limit = 3`, `num_chains = 10`, and `num_repeats = 100`:

```bash
./evaluation_zero_let_linux_parallel 3 10 100
```


## Optional Plotting

The C runtime evaluator writes SVG plots directly. To additionally generate PNG
plots from a runtime CSV file:

```bash
python plot_zero_let_csv.py data_c/data_zero_let_RC_n6_100_12345_20260720_120000.csv
```

Use the actual CSV file name generated in `data_c/`.
